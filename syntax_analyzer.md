# Phase 2: Syntax Analyzer (Parser)

## 1. Definition & Theory
The **Syntax Analyzer**, commonly known as the **Parser**, takes the stream of tokens generated by the Lexer and verifies that they follow the rules of the language's grammar. 

The output of this phase is an **Abstract Syntax Tree (AST)**, which represents the logical structure of the program. If the grammar is violated (e.g., a missing semicolon), the Parser reports a syntax error.

---

## 2. Implementation: Recursive Descent
This project uses a **Top-Down Recursive Descent Parser**. This is a predictive parsing technique where each non-terminal in the grammar is implemented as a separate function.

### **Core Parser Methods**
- `peek()`: Inspects the current token without moving forward.
- `advance()`: Consumes the current token and moves to the next one.
- `matchLex(lex)`: Consumes the token if its text matches `lex`.
- `expectLex(lex)`: Consumes the token or throws an error if it doesn't match.

---

## 3. Handling Operator Precedence
To ensure that mathematical operations are evaluated in the correct order (e.g., multiplication before addition), the parser uses a layered function hierarchy:

```javascript
parseExpression() { return this.parseLogical(); }

parseLogical() {
    let node = this.parseRelational();
    // Handles && and ||
}

parseRelational() {
    let node = this.parseAdditive();
    // Handles <, >, ==, !=
}

parseAdditive() {
    let node = this.parseTerm();
    // Handles + and -
}

parseTerm() {
    let node = this.parseFactor();
    // Handles * and /
}

parseFactor() {
    // Handles (Expression), Numbers, and Identifiers
}
```

---

## 4. AST Node Structure
The resulting AST is a tree where each node describes a high-level construct. 

**Example: `if (a > 10) { print(a); }`**
The AST would look like this conceptually:
- **Node**: `If`
  - **Condition**: `Relational (>)`
    - **Left**: `Identifier (a)`
    - **Right**: `IntConst (10)`
  - **Body**: `Block`
    - **Statement**: `Print`
      - **Argument**: `Identifier (a)`

---

## 5. Error Detection
The Parser detects structural errors by validating "Expectations". If a keyword like `if` is not followed by a `(`, the `expectLex` method triggers an error:
```javascript
expectLex(lex) {
    const t = this.matchLex(lex);
    if (!t) throw new CompilerError(
        `Expected '${lex}'`, 
        this.peek()?.line || 'EOF', 
        this.peek()?.col || 'EOF'
    );
    return t;
}
```
